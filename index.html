HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • meta elements in the head specify meta data such as charset and http-equiv.
  • title defines webpage name which youll find on the tab.
  • h1, h2, h.. define headings.
  • ul, ol and li specify unordered list, ordered list and list item respectively.
  • header, main and footer elements make up the top, middle and bottom of the page respectively.
  • img elements display images that are sourced from the location specified by the src attribute.

CSS

  • CSS can be used inline
  • With an internal CSS style sheet in the header
  • Or using an external CSS style sheet
  • >link rel="stylesheet" href="./assets/style.css"
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Create ./assets/style.css
  • CSS Rule: selector {property: value}
  • * is a catch all selector wildcard
  • CSS Box Model: Margin>Border>Padding>Content
  • We can tag specific elements in HTML with a class so as to apply different CSS rules
  • To specify classes as selectors we need a period to precede the class name
  • >For example: .card { width: 80%; }

Git

  • git status: checks which branch we are currently on
  • git checkout -b branch_name: creates a new branch and switches to it
  • git add -A: adds changes to the staging area so they can be commited
  • git commit -m "your message": commits changes from staging area with a message
  • git pull origin main: pulls the latest main branch from remote repo
  • git push origin branch_name: push changes to the remote repo
  • Next we merge the feature branch through the GitHub interface
  • Confirm changes to the branch
  • Navigate to "Pull Requests" tab and open a new pull request
  • Make sure to select main as the base branch and feature branch as the comparison branch
  • Give PR title and short description
  • Click "Merge pull request"

JavaScript

  • Create a file with the .js type
  • "Link" the .js by adding a script element at the bottom of the body
  • script src="./assets/script.js"
  • A variable is a named container that allows us to store data in our code
  • Declare a variable with: var varName = value;
  • Declare arrays with: var varName = [value1, value2 ..];
  • Arrays are zero-indexed
  • Array length can be accessed with arrayName.length
  • Control flow is the order in which a computer executes code in a script
  • We can alter control flow using conditonal logic (if) and loops (for)
  • if (condition) { // code block }
  • for (var x = 0; x < max; x++) { // code block }
  • Use console.log("message") to print to the console
  • Access the console in Chrome Dev Tools with Ctrl+Shift+I
  • Functions must be defined and called
  • function funcName(parameter1, parameter2 ..) { // code block }
  • funcName(parameter1, parameter2 ..);
  • Generate psudorandom number:
  • Math.floor(Math.random() * arrayName.length)
